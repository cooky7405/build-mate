# 권한별 대시보드 와이어프레임

## 1. 공통 레이아웃 구조

모든 권한의 대시보드는 다음과 같은 기본 레이아웃을 공유합니다:

```
+--------------------------------------------------------------+
|                         헤더 영역                            |
| [로고] [현재 페이지 타이틀]         [검색] [알림] [프로필]   |
+--------------------------------------------------------------+
|          |                                                   |
|          |                                                   |
|  사이드  |                                                   |
|   바     |                메인 콘텐츠 영역                   |
| (네비게  |                                                   |
| 이션)    |                                                   |
|          |                                                   |
+--------------------------------------------------------------+
|                         푸터 영역                            |
| [회사 정보] [도움말] [버전 정보]        [저작권 정보]        |
+--------------------------------------------------------------+
```

사이드바는 모바일 화면에서는 햄버거 메뉴로 전환됩니다.

## 6. 모바일 레이아웃 (공통)

모바일 화면에서는 다음과 같은 레이아웃으로 변환됩니다:

```
+--------------------------------------+
|              헤더 영역               |
| [햄버거 메뉴] [타이틀] [알림][프로필]|
+--------------------------------------+
|                                      |
|                                      |
|           주요 정보 카드 1           |
|                                      |
|                                      |
+--------------------------------------+
|                                      |
|                                      |
|           주요 정보 카드 2           |
|                                      |
|                                      |
+--------------------------------------+
|                                      |
|                                      |
|           주요 정보 카드 3           |
|                                      |
|                                      |
+--------------------------------------+
|                                      |
|        네비게이션 하단 바            |
| [아이콘1] [아이콘2] [아이콘3] [더보기]|
+--------------------------------------+
```

모바일 화면에서는 각 권한별 대시보드의 가장 중요한 정보만 우선적으로 보여주고, 스크롤을 통해 추가 정보에 접근할 수 있도록 설계합니다.

## 7. 대시보드 구현 시 고려사항

1. **반응형 디자인**: 모든 와이어프레임은 다양한 화면 크기에 맞게 조정되어야 함
2. **위젯 커스터마이징**: 사용자가 대시보드 위젯을 추가/제거/재배치할 수 있는 기능 제공
3. **데이터 필터링**: 모든 데이터 표시 영역에는 필터링 옵션이 있어야 함
4. **드릴다운 기능**: 요약 데이터에서 세부 정보로 쉽게 이동할 수 있는 인터페이스 제공
5. **실시간 업데이트**: 중요 정보는 자동 새로고침 또는 알림 기능 구현
6. **접근성**: 모든 UI 요소는 웹 접근성 지침을 준수해야 함
7. **로딩 상태**: 데이터 로딩 중에는 스켈레톤 UI 또는 로딩 인디케이터 표시
